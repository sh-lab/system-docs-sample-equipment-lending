# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS402_lending-request_service`
- 名称：`貸出申請サービス`

## 2. 役割と責務
- 利用者貸出申請・返却画面(V400) における貸出申請実行ユースケースの境界を担う。
- 入力受付、トランザクション境界管理、例外の分類を担当する。
- 対象備品ID群から備品エンティティ一覧への変換を `備品ID指定取得サービス(SQS001)` に委譲する。
- 実際の状態変更と整合性検証は `貸出申請登録コマンドサービス(SCS001)` に委譲する。

## 3. 目的・スコープ
- 目的：利用者が選択した備品に対する貸出申請を登録し、承認待ち状態の申請として受け付けるとともに、対象備品を貸出申請中状態へ更新する。
- スコープ：
  - 対象画面：`利用者貸出申請・返却画面(V400)`
  - 対象データ：貸出申請、貸出申請明細、対象備品
  - 業務範囲：貸出申請登録ユースケースの流れ制御

## 4. 前提条件・事後条件

### 4.1 前提条件
- 認証済みの利用者本人であること。
- 入力値の型・必須・形式検証は Controller 層で完了している前提とする。
- 対象備品IDは 1 件以上であり、同一申請内で重複していないこと。

### 4.2 事後条件
- 正常終了時：
  - 貸出申請が新規登録される。
  - 申請状態は `承認待ち(PENDING_APPROVAL)` となる。
  - 対象備品の状態は `貸出申請中(PENDING_LENDING)` となる。
- 異常終了時：
  - トランザクションはロールバックされる。
  - UI へ業務例外またはシステム例外が通知される。

## 5. 処理フロー概要
1. 利用者ID、対象備品ID群、申請コメントを受け取る。
2. `備品ID指定取得サービス(SQS001)` を用いて、対象備品ID群に対応する備品エンティティ一覧を取得する。
3. 取得した備品エンティティ一覧を `貸出申請登録コマンドサービス(SCS001)` へ引き渡し、登録処理を委譲する。
4. 例外発生時は `02_architecture/error-handling.md` に従い業務例外とシステム例外を分類する。

## 6. 内部で使用するサービス
- Query Service：
  - `備品ID指定取得サービス(SQS001)`
- Command Service：
  - `貸出申請登録コマンドサービス(SCS001)`
- Pure Service：
  - なし

## 7. 入出力

### 7.1 入力
| 項目 | 型 | 必須 | 備考 |
|-----|----|-----|-----|
| userId | string | ○ | 申請者の利用者ID |
| equipmentIds | `List<long>` | ○ | 貸出申請対象の備品ID群。1件以上、重複不可 |
| requestComment | string | 任意 | 最大500文字 |

### 7.2 出力
- 外部公開メソッドの戻り値は `void` とする。

## 8. 例外マッピング方針
- 業務例外：対象備品が 0 件、重複を含む、存在しない、または貸出可能でない場合は申請不可として通知する。
- システム例外：永続化失敗や想定外障害は汎用エラーとして通知する。
- 参照：`02_architecture/error-handling.md`

## 9. トランザクション・整合性
- トランザクション境界は Application Service とする。
- `貸出申請登録コマンドサービス(SCS001)` は本サービスのトランザクション境界を引き継いで実行する。
- 貸出申請本体、貸出申請明細、対象備品状態の更新は 1 つのトランザクションで完結させる。
