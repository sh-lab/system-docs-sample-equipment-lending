# 画面設計書

## 1. 画面ID・名称
- 画面ID：`HFP-EL-V600_admin-equipment-search`
- 画面名称：`管理者備品検索画面`

---

## 2. 概要
本画面は、管理者が登録済み備品を検索し、
新規備品登録および備品情報更新の対象選定を行うための画面である。

利用者向けの `備品検索画面(V300)` とは目的が異なり、
管理者による台帳確認、状態確認、登録時期確認を主目的とする。
そのため、`システム登録日` を含む管理用検索条件および結果表示項目を扱う。

---

## 3. 画面レイアウト概要
- 種別：検索一覧・導線画面
- 主な構成：
  - 上部：画面タイトル、メッセージエリア
  - 中央上段：検索条件
  - 中央下段：検索結果一覧
  - 下部：管理者マイページへ戻るボタン、新規備品登録ボタン

### レイアウトイメージ
```text
+------------------------------------------------------------------------------------------------------+
|                                       管理者備品検索画面                                             |
+------------------------------------------------------------------------------------------------------+
| [i] 該当する備品はありません。                                                                       |
| [!] 検索結果が100件を超えています。先頭100件を表示します。条件を絞ってください。                     |
|                                                                                                      |
| 検索条件                                                                                              |
| 備品名 [______________] 備品種別 [▼すべて] 備品状態 [▼すべて] システム登録日 [____-__-__] [検索]    |
|                                                                                                      |
| 検索結果                                                                                              |
| +--------+------------+--------------+------------------+----------------+------------+------------+ |
| | 状態更新 | 備品管理番号 | 備品名       | 備品種別         | システム登録日 | 保管場所   | 備品状態   | |
| +--------+------------+--------------+------------------+----------------+------------+------------+ |
| | [変更] | EQ-0001    | 長机 2台      | 長机             | 2026-03-01     | 第1倉庫    | 貸出可能   | |
| | [変更] | EQ-0100    | ライト        | ライト           | 2026-03-20     | 備品庫     | 貸出不可   | |
| +--------+------------+--------------+------------------+----------------+------------+------------+ |
|                                                                                                      |
| [管理者マイページへ戻る] [新規備品登録]                                                               |
+------------------------------------------------------------------------------------------------------+
```

### 表示方針
- 画面タイトルとして `管理者備品検索画面` を表示する。
- 初期表示では条件未指定で検索し、登録済み備品を一覧表示する。
- 検索条件には `備品名`、`備品種別`、`備品状態`、`システム登録日` を表示する。
- `備品種別` の候補は備品種別マスタから取得し、画面上で追加・変更しない。
- 検索結果には、`備品管理番号`、`備品名`、`備品種別`、`システム登録日`、`保管場所`、`備品状態` を表示する。
- 各行の `状態更新` 操作から、対象備品を読み込んだ `管理者備品編集画面` へ遷移できるものとする。
- `新規備品登録` ボタンから、新規登録モードの `管理者備品編集画面` へ遷移できるものとする。
- 検索結果は `備品管理番号` 昇順で表示する。
- 検索結果が 101 件以上の場合は先頭 100 件のみ表示し、ワーニングメッセージを表示する。
- 検索結果が 0 件の場合はインフォメッセージを表示する。

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | 画面表示時 | 条件未指定で備品を検索し、先頭100件を表示する | `管理者備品検索初期表示サービス(SAS601)` |
| 検索実行 | `[検索]` ボタンクリック | 入力された検索条件で備品を検索し、先頭100件を表示する | `管理者備品検索実行サービス(SAS602)` |

## 6. バリデーション方針
### クライアント側
- `備品名` は最大100文字とし、超過入力は受け付けない。
- `備品種別`、`備品状態` は定義済み候補から選択する。
- `備品種別` の候補は備品種別マスタから取得し、画面上で追加・変更しない。
- `システム登録日` は日付入力のみ受け付ける。

### サーバ側
- 本画面は `ADMIN` 権限を持つ認証済み利用者を利用対象とし、認証・認可の具体設計は `docs/03_designs/security-design.md` に従う。
- 検索結果件数判定のため、内部的には 101 件目まで取得し、表示は先頭 100 件までに制限する。
- `システム登録日` は完全一致条件として扱う。
- 管理者備品編集画面への遷移時は、対象備品IDが存在することを後続画面で再検証する。

---

## 7. エラー・メッセージ表示方針
- 検索結果が 101 件以上の場合：ワーニング「検索結果が100件を超えています。先頭100件を表示します。条件を絞ってください。」を表示する。
- 検索結果が 0 件の場合：インフォ「該当する備品はありません。」を表示する。
- 指定したシステム登録日が不正な場合：業務メッセージを表示し、入力の見直しを促す。
- システム例外：汎用エラー画面へ遷移する。

`02_architecture/error-handling.md` に従い、UI 層では業務例外のみユーザー通知し、システム例外は汎用エラーとして扱う。

---

## 8. 非機能要件への配慮
- 表示性能：一覧表示は最大100件に制限し、件数超過時は絞り込みを促す。
- 操作性：検索、状態確認、状態更新開始、新規備品登録開始を1画面で完結できる構成とする。
- アクセシビリティ：キーボード操作のみで検索条件入力、検索実行、一覧操作、戻る操作へ到達できる構成とする。
- 一貫性：利用者向けの `備品検索画面(V300)` と分離し、管理業務向けの検索条件と結果項目を明確にする。
