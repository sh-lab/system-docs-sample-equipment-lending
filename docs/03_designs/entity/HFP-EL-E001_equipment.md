# エンティティ設計書（Entity Design）

## 1. エンティティID・名称
- エンティティID：`HFP-EL-E001_equipment`
- 名称：`Equipment / 備品`

---

## 2. 目的・スコープ
- 目的：
  - 本エンティティは、貸出対象となる備品の識別情報、検索条件、現在状態を管理するための永続化単位である。
- スコープ：
  - 本エンティティが扱う情報
    - 備品の管理番号
    - 備品名
    - 備品種別
    - 保管場所
    - 現在の貸出関連状態
    - 備考
  - 本エンティティでは扱わない情報
    - 貸出申請・返却申請そのものの履歴
    - 承認者コメントや却下理由
    - 貸出期間、返却日時など申請トランザクションに属する情報
- 参照：
  - 関連ユースケース：`UC-001`、`UC-002`、`UC-003`、`UC-004`、`UC-005`
  - 関連機能要件：`FR-001`、`FR-002`、`FR-003`、`FR-004`、`FR-005`
  - 関連アプリケーションサービス：後続設計で定義する

---

## 3. 構造・境界
本エンティティは、備品1件を1つの本体エンティティとして扱う。

- 本体エンティティとして扱う要素
  - 備品を一意に識別する情報
  - 備品検索に必要な基本属性
  - 貸出・返却業務で参照される現在状態
- 本体エンティティに付随する要素
  - なし
- 本エンティティとしての操作および管理の単位
  - 備品1件を登録・更新・参照の単位とする
  - 貸出申請や返却申請による状態変更は、備品単体ではなく申請系ユースケースから間接的に行う

---

## 4. 属性（意味論）
> 物理名・型・索引・テーブル定義は table-design に記載する。

### 4.0 本体エンティティの属性

| 属性名（英） | 表示名 | 必須 | 不変/可変 | 意味・説明 |
|--------------|--------|------|-----------|------------|
| id | 備品ID | 必須 | 不変 | 備品を一意に識別する内部ID |
| equipmentCode | 備品管理番号 | 必須 | 不変 | 運用上の備品識別子。検索・照会に用いる |
| name | 備品名 | 必須 | 可変 | 利用者・管理者に提示する備品名称 |
| equipmentType | 備品種別 | 必須 | 可変 | 検索条件および分類に用いる粗い種別。数量差は備品名で表現する |
| storageLocation | 保管場所 | 必須 | 可変 | 備品の通常保管場所 |
| status | 備品状態 | 必須 | 可変 | `貸出可能`、`貸出申請中`、`貸出中`、`貸出不可`、`廃棄` のいずれかを表す現在状態 |
| remarks | 備考 | 任意 | 可変 | 補足事項。画面表示必須ではない |

---

## 5. 不変条件（Invariants）

### 本体エンティティ
- 備品ID は一意であり変更不可である。
- 備品管理番号は一意であり変更不可である。
- 備品状態は定義済みの値のみを許可する。
- `貸出申請中`、`貸出中`、`貸出不可`、`廃棄` の状態にある備品は、貸出申請対象として扱ってはならない。
- 保管場所は通常時の所在を示す属性であり、申請処理のたびに変更してはならない。

---

## 6. 状態モデル
- 許可される遷移：
  - `貸出可能`（`AVAILABLE`）→ `貸出申請中`（`PENDING_LENDING`）
    - 貸出申請の登録により遷移する。
  - `貸出申請中`（`PENDING_LENDING`）→ `貸出中`（`LENT`）
    - 貸出申請の承認確定により遷移する。
  - `貸出申請中`（`PENDING_LENDING`）→ `貸出可能`（`AVAILABLE`）
    - 貸出申請の却下確定により遷移する。
  - `貸出中`（`LENT`）→ `貸出可能`（`AVAILABLE`）
    - 返却確認の確定により遷移する。
- 禁止される遷移：
  - `貸出可能`（`AVAILABLE`）→ `貸出不可`（`UNAVAILABLE`）
  - `貸出可能`（`AVAILABLE`）→ `廃棄`（`DISPOSED`）
  - `貸出申請中`（`PENDING_LENDING`）→ `貸出不可`（`UNAVAILABLE`）
  - `貸出申請中`（`PENDING_LENDING`）→ `廃棄`（`DISPOSED`）
  - `貸出中`（`LENT`）→ `貸出不可`（`UNAVAILABLE`）
  - `貸出中`（`LENT`）→ `廃棄`（`DISPOSED`）
  - `貸出不可`（`UNAVAILABLE`）を起点とする遷移
  - `廃棄`（`DISPOSED`）を起点とする遷移

補足：
- 現行スコープで業務上使用する状態は `貸出可能`、`貸出申請中`、`貸出中` である。
- `貸出不可` および `廃棄` は次期開発用の予約状態であり、本版では遷移を定義しない。
- 申請の進行状態は申請系エンティティ側で表現する。
- 備品検索画面では、`貸出可能` を除く状態を表示上 `貸出不可` として扱ってよい。

## 8. 派生値・計算ロジック
- 貸出申請可否の判定（`status == AVAILABLE` かどうか）は、備品エンティティの責務ではなく、ピュアサービス（SPS001）が担う。
- `貸出申請中`、`貸出中`、`貸出不可`、`廃棄` は、いずれも貸出不可として判定する。

---

## 9. 操作・設計上の制約
- 備品の生成・更新・状態変更は、アプリケーションサービス経由でのみ行う。
- 備品状態の直接更新は、対応するユースケースの前提条件・状態遷移に従わなければならない。
- 備品エンティティは単独の明細要素を持たない。
- 初回設計では、備品種別・保管場所・状態コードを別マスタへ分離しない。
- 実装上の備品エンティティは `src/main/java/net/shlab/hogefugapiyo/equipmentlending/model/entity/Equipment.java` に配置する。
- 実装クラスは `AuditVersionEntity` を継承する通常クラスとして定義し、`record` は用いない。
- 備品状態は `src/main/java/net/shlab/hogefugapiyo/equipmentlending/model/value/EquipmentStatus.java` の enum で表現する。
- 備品用 Entity Repository は Spring Data JPA および `EntityManager` を用いて実装する。

---

## 10. 監査・バージョニング
- 監査情報はシステムが自動設定する。
- 併存更新は JPA の `@Version` による楽観ロックで制御する。
- 状態変更が正常終了した場合の操作履歴は `H_EQUIPMENT_HISTORY` に記録する。
- 操作履歴はエンティティ本体の業務属性ではなく、`operationId` 単位の追跡情報として別テーブルで管理する。
- 永続化は `EntityRepository` を継承した備品用 Repository を通じて行う。
