# 画面設計書

## 1. 画面ID・名称
- 画面ID：`HFP-EL-V400_user-lending-request`
- 画面名称：`利用者貸出申請・返却画面`

---

## 2. 概要
本画面は、利用者が備品検索画面で選択した備品を確認して貸出申請を登録し、あわせて自身の貸出中申請に対して返却申請を行うための画面である。

また、利用者マイページから遷移した場合は、却下された申請内容の確認も同一画面で扱い、状態に応じて利用可能な操作のみを表示する。V400 は単一レイアウトをベースとし、`貸出申請`、`返却申請`、`却下確認` の3つの表示モードを持つ。

---

## 3. 画面レイアウト概要
- 種別：入力・詳細画面
- 主な構成：
  - 上部：画面タイトル、モード別メッセージ
  - 中央上段：申請情報（申請ID、状態、各種日時）
  - 中央中段：対象備品一覧
  - 中央下段：コメント表示／入力領域
  - 下部：戻るボタン、モード別実行ボタン

### 表示方針
- 遷移元が `備品検索画面(V300)` の場合は `貸出申請` とし、選択済み備品一覧と申請コメント入力欄を表示する。
- 遷移元が `利用者マイページ(V100)` かつ対象申請状態が `LENT` の場合は `返却申請` とし、既存申請情報と返却申請コメント入力欄を表示する。
- 遷移元が `利用者マイページ(V100)` かつ対象申請状態が `REJECTED` の場合は `却下確認` とし、申請内容と管理者コメントを参照表示し、確認完了操作のみを提供する。
- 申請情報に表示する日時は、存在する項目のみ表示する。未設定項目は表示しない。
- 対象備品一覧は、申請対象の全備品を `備品管理番号` 昇順で表示する。
- 貸出申請では `備品検索へ戻る` を表示する。返却申請および却下確認では `マイページへ戻る` を表示する。
- 実行ボタンのラベルはモードに応じて `貸出申請する`、`返却申請する`、`確認済みにする` を切り替える。

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | 画面表示時 | 遷移元と対象データに応じて画面モードを判定し、申請情報・対象備品一覧・入力可能項目を組み立てて表示する | `利用者貸出申請・返却画面初期表示サービス(SAS401)` |
| 貸出申請実行 | `[貸出申請する]` ボタンクリック | 選択済み備品一覧と申請コメントを受け取り、貸出申請を登録する。完了後は利用者マイページへ遷移し、完了メッセージを表示する | `貸出申請サービス(SAS402)` |
| 返却申請実行 | `[返却申請する]` ボタンクリック | 対象申請と返却申請コメントを受け取り、返却申請を登録する。完了後は利用者マイページへ遷移し、完了メッセージを表示する | `返却申請サービス(SAS403)` |
| 却下確認完了 | `[確認済みにする]` ボタンクリック | 却下状態の申請に対する確認完了を登録し、申請を完了状態へ更新する。完了後は利用者マイページへ遷移し、完了メッセージを表示する | `却下確認サービス(SAS404)` |
| 備品検索へ戻る | `[備品検索へ戻る]` ボタンクリック | 備品検索画面へ戻る。再選択を可能にする | `－（画面遷移のみ）` |
| マイページへ戻る | `[マイページへ戻る]` ボタンクリック | 利用者マイページへ戻る | `－（画面遷移のみ）` |

## 6. バリデーション方針
### クライアント側
- `申請コメント` および `返却申請コメント` は最大500文字とし、超過入力は受け付けない。
- 貸出申請では、対象備品が 1 件以上保持されていない場合、`貸出申請する` を非活性表示とする。
- 返却申請および却下確認では、対象モード以外の入力項目を表示しない。
- 実行ボタン押下後は、二重送信防止のためボタンを一時非活性化する。

### サーバ側
- 本画面は `USER` 権限を持つ認証済み利用者を利用対象とし、認証・認可の具体設計は `docs/03_designs/security-design.md` に従う。
- 貸出申請、返却申請、却下確認は、認証済み利用者本人のみ実行可能とする。
- 貸出申請初期表示時は、引き継がれた対象備品IDが 1 件以上であり、重複を含まず、全件が存在することを確認する。
- 貸出申請実行時は、`貸出申請サービス(SAS402)` が `備品ID指定取得サービス(SQS001)` により対象備品エンティティ一覧を再取得し、その結果を `貸出申請登録コマンドサービス(SCS001)` へ引き渡す。
- `貸出申請登録コマンドサービス(SCS001)` は、受け取った対象備品エンティティ一覧を `貸出申請可否判定サービス(SPS001)` で再検証し、対象備品が現在もすべて `AVAILABLE` であることを確認する。`PENDING_LENDING`、`LENT`、`UNAVAILABLE`、`DISPOSED` の備品が含まれる場合は業務例外とする。
- 返却申請実行時は、`返却申請登録コマンドサービス(SCS002)` が対象申請の所有者、現在状態 `LENT`、`VERSION` の一致を確認する。
- 却下確認完了時は、`却下確認完了コマンドサービス(SCS003)` が対象申請の所有者、現在状態 `REJECTED`、`VERSION` の一致を確認する。
- 対象備品一覧は 1 件以上でなければならず、同一申請内の重複備品は許可しない。

## 7. エラー・メッセージ表示方針
- 貸出申請成功時：利用者マイページでインフォ「貸出申請を受け付けました。」を表示する。
- 返却申請成功時：利用者マイページでインフォ「返却申請を受け付けました。」を表示する。
- 却下確認完了時：利用者マイページでインフォ「却下内容の確認が完了しました。」を表示する。
- 対象備品の状態が変化して申請できない場合：画面上部に業務メッセージ「備品の状態が変わりました。再度選択してください。」を表示し、備品検索画面での再選択を促す。
- 返却対象申請が貸出中でない場合：画面上部に業務メッセージを表示し、マイページへの再遷移を促す。
- 却下対象申請が既に確認済みである場合：画面上部に業務メッセージを表示し、マイページの再表示を促す。
- 楽観ロック不一致時：画面上部に業務メッセージを表示し、最新状態の再確認を促す。
- システム例外：汎用エラー画面へ遷移する。

`02_architecture/error-handling.md` に従い、UI層では業務例外のみユーザー通知し、システム例外は汎用エラーとして扱う。

## 8. 非機能要件への配慮
- 操作性：共通レイアウトを維持しつつ、モードごとに操作対象と入力可能項目を限定し、利用者が現在行うべき業務を把握しやすくする。
- 重複操作防止：更新系ボタン押下時は二重送信を防止し、同一申請に対する重複更新を抑止する。
- アクセシビリティ：キーボード操作のみでコメント入力、対象備品一覧確認、戻る操作、実行操作へ到達できる構成とする。
- セキュリティ：更新系操作は CSRF トークン送信を前提とし、他人の申請IDを指定した場合は業務例外または認可エラーとして扱う。
- 一貫性：申請状態と利用可能操作の対応は `HFP-EL-E003_lending-request` の状態遷移定義に従う。

## 9. 備考
- 本画面は利用者向け画面であり、管理者向けの承認・返却確認操作は `管理者承認・却下・返却確認画面(V500)` で扱う。
- 貸出申請への遷移元は `備品検索画面(V300)`、返却申請および却下確認への遷移元は `利用者マイページ(V100)` である。
- `利用者マイページ(V100)` からの詳細遷移は、申請状態に応じて返却申請または却下確認へ分岐する。
- 申請詳細は `T_LENDING_REQUEST` と `T_LENDING_REQUEST_DETAIL` を単位として参照し、備品の表示用情報は `M_EQUIPMENT` および `M_EQUIPMENT_TYPE` から補完する。
- 別権限の認証済み利用者が本画面へアクセスした場合の制御は、`docs/03_designs/security-design.md` に従う。

- 関連ユースケース：`UC-002`、`UC-004`、`UC-006`
