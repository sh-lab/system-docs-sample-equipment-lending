# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS403_return-request_service`
- 名称：`返却申請サービス`

## 2. 役割と責務
- 利用者貸出申請・返却画面(V400) における返却申請実行ユースケースの境界を担う。
- 入力受付、トランザクション境界管理、例外の分類を担当する。
- 実際の状態更新と整合性検証は `返却申請登録コマンドサービス(SCS002)` に委譲する。

## 3. 目的・スコープ
- 目的：貸出中の申請に対して返却申請を登録し、返却確認待ち状態へ遷移させる。
- スコープ：
  - 対象画面：`利用者貸出申請・返却画面(V400)`
  - 対象データ：貸出申請
  - 業務範囲：返却申請登録ユースケースの流れ制御

## 4. 前提条件・事後条件

### 4.1 前提条件
- 認証済みの利用者本人であること。
- 入力値の型・必須・形式検証は Controller 層で完了している前提とする。
- 対象申請IDおよび楽観ロック用バージョンが入力されていること。

### 4.2 事後条件
- 正常終了時：
  - 対象申請の状態が `返却確認待ち(PENDING_RETURN_CONFIRMATION)` へ更新される。
  - 返却申請コメントおよび返却申請日時が記録される。
- 異常終了時：
  - トランザクションはロールバックされる。
  - UI へ業務例外またはシステム例外が通知される。

## 5. 処理フロー概要
1. 利用者ID、貸出申請ID、返却申請コメント、バージョンを受け取る。
2. `返却申請登録コマンドサービス(SCS002)` に更新処理を委譲する。
3. 例外発生時は `02_architecture/error-handling.md` に従い業務例外とシステム例外を分類する。

## 6. 内部で使用するサービス
- Query Service：
  - なし
- Command Service：
  - `返却申請登録コマンドサービス(SCS002)`
- Pure Service：
  - なし

## 7. 入出力

### 7.1 入力
| 項目 | 型 | 必須 | 備考 |
|-----|----|-----|-----|
| userId | string | ○ | 操作対象の利用者ID |
| lendingRequestId | long | ○ | 返却申請対象の貸出申請ID |
| returnRequestComment | string | 任意 | 最大500文字 |
| version | long | ○ | 楽観ロック確認用 |

### 7.2 出力
- 外部公開メソッドの戻り値は `void` とする。

## 8. 例外マッピング方針
- 業務例外：対象申請が存在しない、本人以外の申請である、貸出中でない、またはバージョン不一致の場合は更新不可として通知する。
- システム例外：永続化失敗や想定外障害は汎用エラーとして通知する。
- 参照：`02_architecture/error-handling.md`

## 9. トランザクション・整合性
- トランザクション境界は Application Service とする。
- `返却申請登録コマンドサービス(SCS002)` は本サービスのトランザクション境界を引き継いで実行する。
- 対象申請の取得、状態確認、返却申請更新は 1 つのトランザクションで行う。
