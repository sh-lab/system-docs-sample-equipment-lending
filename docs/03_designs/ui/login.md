# 画面設計書

## 1. 画面ID・名称
- 画面ID：`login`
- 画面名称：`ログイン画面`

---

## 2. 概要
本画面は、未認証の利用者または管理者がユーザーIDとパスワードを入力し、認証後に利用者区分に応じたマイページへ遷移するための基盤画面である。

本画面では認証入力欄に加え、学習用途に限り、試験用利用者としてあらかじめ定義された利用者一覧を固定表示し、利用者がログインに用いるIDを確認できるようにする。

---

## 3. 画面レイアウト概要
- 種別：入力画面
- 主な構成：
  - 上部：画面タイトル、説明文
  - 中央：エラーメッセージ、ログインフォーム
  - 下部：試験用パスワード案内、試験用利用者一覧

### レイアウトイメージ
```text
+------------------------------------------------+
|                  ログイン                      |
+------------------------------------------------+
| ユーザーIDとパスワードを入力してください。     |
|                                                |
| [エラーメッセージ]                             |
| ※ 認証失敗時のみ表示                          |
|                                                |
| ユーザーID [              ]                    |
| パスワード [              ]                    |
| [ログイン]                                     |
|                                                |
| 試験用パスワードは `pass` です。               |
|                                                |
| 試験用利用者一覧                               |
| +------------+----------+------------+         |
| | ユーザーID | 氏名     | 利用者区分 |         |
| +------------+----------+------------+         |
| | USER01     | 利用者01 | 利用者     |         |
| | USER02     | 利用者02 | 利用者     |         |
| | USER03     | 利用者03 | 利用者     |         |
| | USER04     | 利用者04 | 利用者     |         |
| | ADMIN1     | 管理者01 | 管理者     |         |
| | ADMIN2     | 管理者02 | 管理者     |         |
| +------------+----------+------------+         |
+------------------------------------------------+
```

### 表示方針
- 試験用利用者一覧は、学習用途に限って表示する補助情報とする。
- 試験用利用者一覧には、あらかじめ定義された利用者のユーザーID、氏名、利用者区分を固定表示する。
- 試験用利用者一覧には、パスワードは表示せず、共通パスワード案内を別行で表示する。
- エラーメッセージ表示時も、ログインフォーム入力値と試験用利用者一覧を再表示する。

## 4. 画面項目定義
| 項目名 | 種別 | 必須 | 入力制約 | 表示形式 | 備考 |
|------|------|------|---------|---------|------|
| 画面タイトル | ラベル | - | - | テキスト | `ログイン` を固定表示する |
| 説明文 | メッセージ | - | - | テキスト | `ユーザーIDとパスワードを入力してください。` を表示する |
| エラーメッセージ | メッセージ | - | - | エラー表示 | 認証失敗時に表示する |
| ユーザーID | テキスト入力 | ○ | 最大6文字 | テキスト | 未入力または存在しないIDは認証失敗として扱う |
| パスワード | パスワード入力 | ○ | 最大20文字 | マスク入力 | 試験用共通パスワードを入力する |
| ログイン | ボタン | ○ | - | ボタン | 押下でログイン処理を実行する |
| 試験用パスワード案内 | 注記 | - | - | テキスト | `試験用パスワードは pass です。` を表示する |
| 試験用利用者一覧 | テーブル | - | - | 一覧表示 | 学習用途に限って、あらかじめ定義された試験用利用者一覧を固定表示する |
| 試験用利用者一覧.ユーザーID | テキスト | - | 6文字固定 | テキスト | 試験用利用者の識別子を表示する |
| 試験用利用者一覧.氏名 | テキスト | - | 最大100文字 | テキスト | 試験用利用者の氏名を表示する |
| 試験用利用者一覧.利用者区分 | ラベル | - | `USER` / `ADMIN` | ラベル | 利用者区分を `利用者` または `管理者` として表示する |

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | 画面表示時 | ログインフォームおよび固定の試験用利用者一覧を表示する | `－（画面表示のみ）` |
| ログイン実行 | `[ログイン]` ボタンクリック | 入力されたユーザーIDとパスワードで認証し、利用者区分に応じたマイページへ遷移する | `LoginApplicationService` |
| 認証失敗再表示 | 認証失敗時 | エラーメッセージ、入力済みユーザーID、固定の試験用利用者一覧を再表示する | `－（画面再表示のみ）` |

---

## 6. バリデーション方針
### クライアント側
- ユーザーIDおよびパスワードはブラウザ標準の入力欄で受け付ける。
- クライアント側で業務固有の認証判定は行わない。

### サーバ側
- `login` 画面は未認証利用者を対象とし、認証済み利用者がアクセスした場合は利用者区分に応じたホーム画面へリダイレクトする。
- ユーザーIDがあらかじめ定義された試験用利用者に存在しない場合、またはパスワードが共通パスワードと一致しない場合は認証失敗とする。
- 認証済み利用者であっても、利用者区分に応じて利用可能な画面および機能を制限する。

---

## 7. エラー・メッセージ表示方針
- 認証失敗時：画面上部に認証失敗メッセージを表示する。
- エラーメッセージは利用者に再入力を促す内容とし、詳細な内部情報は表示しない。
- システム例外：汎用エラー画面へ遷移する。

---

## 8. 非機能要件への配慮
- 操作性：学習利用者がIDを転記しやすいよう、試験用利用者一覧をフォーム近傍に配置する。
- 保守性：固定表示する試験用利用者一覧は、認証に利用する定義と整合するよう管理する。
- セキュリティ：試験用利用者一覧には共通パスワード以外の認証情報を表示しない。
