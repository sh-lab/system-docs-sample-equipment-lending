# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS504_return-confirm_service`
- 名称：`返却確認サービス`

## 2. 役割と責務
- 管理者承認・却下・返却確認画面(V500) における返却確認ユースケース境界を担う。
- 入力受付、トランザクション境界管理、例外の分類を担当する。
- 実際の状態更新は `返却確認コマンドサービス(SCS012)` に委譲する。

## 3. 目的・スコープ
- 目的：返却確認待ちの申請を返却確認し、完了状態へ更新するとともに、対象備品を貸出可能状態へ戻す。
- スコープ：
  - 対象画面：`管理者承認・却下・返却確認画面(V500)`
  - 対象データ：貸出申請、対象備品
  - 業務範囲：返却確認ユースケースの流れ制御

## 4. 前提条件・事後条件

### 4.1 前提条件
- 認証済みの管理者であること。
- 入力値の型・必須・形式検証は Controller 層で完了している前提とする。
- 対象申請IDおよび楽観ロック用バージョンが入力されていること。

### 4.2 事後条件
- 正常終了時：
  - 対象申請の状態が `COMPLETED` へ更新される。
  - 返却確認者、返却確認コメント、完了日時が記録される。
  - 対象備品の状態が `AVAILABLE` へ更新される。
- 異常終了時：
  - トランザクションはロールバックされる。
  - UI へ業務例外またはシステム例外が通知される。

## 5. 処理フロー概要
1. 管理者ユーザーID、貸出申請ID、返却確認コメント、バージョンを受け取る。
2. `返却確認コマンドサービス(SCS012)` に更新処理を委譲する。
3. 例外発生時は `02_architecture/error-handling.md` に従い業務例外とシステム例外を分類する。

## 6. 内部で使用するサービス
- Query Service：
  - なし
- Command Service：
  - `返却確認コマンドサービス(SCS012)`
- Pure Service：
  - なし

## 7. 入出力

### 7.1 入力
| 項目 | 型 | 必須 | 備考 |
|-----|----|-----|-----|
| adminUserId | string | ○ | 返却確認を行う管理者ユーザーID |
| lendingRequestId | long | ○ | 返却確認対象の貸出申請ID |
| returnConfirmComment | string | 任意 | 最大500文字 |
| version | long | ○ | 楽観ロック確認用 |

### 7.2 出力
- 外部公開メソッドの戻り値は `void` とする。

## 8. 例外マッピング方針
- 業務例外：対象申請が存在しない、返却確認待ちでない、またはバージョン不一致の場合は更新不可として通知する。
- システム例外：永続化失敗や想定外障害は汎用エラーとして通知する。
- 参照：`02_architecture/error-handling.md`

## 9. トランザクション・整合性
- トランザクション境界は Application Service とする。
- `返却確認コマンドサービス(SCS012)` は本サービスのトランザクション境界を引き継いで実行する。
- 申請更新と対象備品状態更新は 1 つのトランザクションで完結させる。
