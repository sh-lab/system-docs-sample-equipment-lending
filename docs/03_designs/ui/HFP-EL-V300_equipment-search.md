# 画面設計書

## 1. 画面ID・名称
- 画面ID：`HFP-EL-V300_equipment-search`
- 画面名称：`備品検索画面`

---

## 2. 概要
本画面は、利用者が貸出対象となる備品を検索し、貸出申請対象を複数選択して貸出申請画面へ遷移するための画面である。

初期表示では貸出申請可能な備品を表示し、利用者が条件を絞り込みながら候補を選定できるようにする。検索結果は最大 100 件まで表示し、101 件以上が該当する場合はワーニングメッセージを表示して検索条件の再指定を促す。

---

## 3. 画面レイアウト概要
- 種別：検索一覧画面
- 主な構成：
  - 上部：画面タイトル
  - タイトル下：ワーニングメッセージ／インフォメッセージ
  - 中央上段：検索条件
  - 中央下段：検索結果一覧
  - 下部：戻るボタン、貸出申請ボタン

### レイアウトイメージ
```text
+----------------------------------------------------------------------------------+
|                                   備品検索画面                                   |
+----------------------------------------------------------------------------------+
| [!] 検索結果が100件を超えています。先頭100件を表示します。                       |
|     条件を絞ってください。                                                       |
| [i] 該当する備品はありません。                                                   |
| ※ 上記メッセージは条件に応じていずれかを表示する                                 |
|                                                                                  |
| 検索条件                                                                          |
| 備品名 [____________________]  備品種別 [▼すべて]  貸出状態 [▼貸出可能] [検索]   |
|                                                                                  |
| 検索結果                                                                          |
| +----+------------+--------------+------------------+------------+----------+    |
| |選択| 備品管理番号 | 備品名       | 備品種別         | 保管場所   | 状態     |    |
| +----+------------+--------------+------------------+------------+----------+    |
| |[ ] | EQ-0001    | 長机 2台      | 長机             | 第1倉庫    | 貸出可能 |    |
| |[ ] | EQ-0021    | プロジェクター | プロジェクター   | 映像機材庫 | 貸出可能 |    |
| |[-] | EQ-0002    | 長机 2台      | 長机             | 第1倉庫    | 貸出不可 |    |
| +----+------------+--------------+------------------+------------+----------+    |
| ※ 貸出不可の行は選択不可                                                         |
|                                                                                  |
| [マイページへ戻る]  [貸出申請]                                                   |
+----------------------------------------------------------------------------------+
```

### 表示方針
- 初期表示では、`貸出状態 = 貸出可能` の条件で検索した結果を表示する。
- 検索条件未入力時も、初期表示と同様に貸出申請可能な備品を表示する。
- 検索結果は `備品管理番号` 昇順で表示する。
- 検索結果一覧の左端に複数選択用チェックボックスを配置し、`貸出可能` の行のみ選択可能とする。
- `貸出不可` の備品を結果に表示する場合も、状態識別のため一覧表示は行うが、チェックボックスは非活性表示とする。
- 本画面における `貸出不可` は、内部状態 `貸出申請中`、`貸出中`、`貸出不可`、`廃棄` をまとめた表示用区分である。
- 検索結果が 101 件以上の場合は、先頭 100 件のみ表示し、画面上部にワーニングメッセージを表示する。
- 検索結果が 0 件の場合は、画面上部にインフォメッセージを表示し、結果一覧は空表示とする。
- `マイページへ戻る` ボタンは常時表示し、押下で利用者マイページへ戻れるようにする。
- `貸出申請` ボタンは、1 件以上選択されている場合のみ活性表示する。

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | 画面表示時 | `貸出状態 = 貸出可能` を初期条件として備品を検索し、先頭100件を表示する。検索結果が101件以上の場合はワーニングメッセージを表示する | `備品検索初期表示サービス(SAS301)` |
| 検索実行 | `[検索]` ボタンクリック | 入力された検索条件で備品を検索し、先頭100件を表示する。検索結果が0件の場合はインフォメッセージ、101件以上の場合はワーニングメッセージを表示する | `備品検索実行サービス(SAS302)` |
| 選択状態更新 | 検索結果一覧のチェックボックス変更 | 選択件数に応じて `貸出申請` ボタンの活性／非活性を切り替える | `－（画面内制御のみ）` |
| マイページへ戻る | `[マイページへ戻る]` ボタンクリック | 利用者マイページへ遷移する | `－（画面遷移のみ）` |
| 貸出申請開始 | `[貸出申請]` ボタンクリック | 選択した備品ID群を受け取り、遷移直前に選択備品が申請可能状態であることを再確認する。全件が申請可能な場合のみ貸出申請画面へ遷移する | `貸出申請開始サービス(SAS303)` |

## 6. バリデーション方針
### クライアント側
- `備品名` は最大100文字とし、超過入力は受け付けない。
- `備品種別`、`貸出状態` は定義済み候補から選択する。
- `貸出不可` の行および選択不可の行は、チェックボックスを非活性表示とする。
- `貸出申請` ボタンは、1件以上の備品が選択されるまで非活性表示とする。
- 検索結果が 0 件の場合でもエラーとはせず、インフォメッセージで案内する。

### サーバ側
- 本画面は `USER` 権限を持つ認証済み利用者を利用対象とし、認証・認可の具体設計は `docs/03_designs/security-design.md` に従う。
- 画面遷移時の利用者識別は、認証済み利用者本人を前提に行う。
- 検索対象の状態は `AVAILABLE`、`PENDING_LENDING`、`LENT`、`UNAVAILABLE`、`DISPOSED` とし、画面表示時には `貸出可能` または `貸出不可` へ正規化する。
- 検索結果件数判定のため、内部的には 101 件目まで取得し、表示は先頭 100 件までに制限する。
- `貸出申請` 開始時は、`貸出申請開始サービス(SAS303)` が `貸出申請可否判定サービス(SPS001)` を呼び出し、選択された全備品が現在も `AVAILABLE` であることを再検証する。
- 選択された備品が 0 件、重複している、存在しない、または申請不可状態の備品を含む場合は、業務例外として扱う。

---

## 7. エラー・メッセージ表示方針
- 検索結果が 101 件以上の場合：画面上部にワーニング「検索結果が100件を超えています。先頭100件を表示します。条件を絞ってください。」を表示する。
- 検索結果が 0 件の場合：画面上部にインフォ「該当する備品はありません。」を表示する。
- 備品未選択で `貸出申請` が実行された場合：画面上部に業務メッセージを表示し、1件以上の選択を促す。
- 選択備品に申請不可状態のものが含まれる場合：画面上部に業務メッセージ「備品の状態が変わりました。再度選択してください。」を表示し、同一検索結果上で再選択を促す。
- システム例外：汎用エラー画面へ遷移する。

`02_architecture/error-handling.md` に従い、UI層では業務例外のみユーザー通知し、システム例外は汎用エラーとして扱う。

---

## 8. 非機能要件への配慮
- 表示性能：一覧表示は最大100件に制限し、件数超過時はワーニングで絞り込みを促す。
- 操作性：検索条件、結果一覧、戻るボタン、貸出申請ボタンを縦方向に配置し、検索から選択、遷移、前画面復帰までの流れを明確にする。
- 重複操作防止：`貸出申請` ボタン押下中は連打による二重遷移を抑止する。
- アクセシビリティ：キーボード操作のみで検索条件入力、結果一覧のチェックボックス選択、貸出申請ボタン操作へ到達できる構成とする。

## 9. 備考
- 本画面は利用者向け画面であり、管理者は利用対象外とする。
- 貸出申請画面の詳細設計は後続で定義する。
- `備品種別` および `状態` の表示名は、`M_EQUIPMENT` の保持値を画面表示用に変換して表示する。
- `状態` は内部状態 `AVAILABLE` を `貸出可能`、`PENDING_LENDING`、`LENT`、`UNAVAILABLE`、`DISPOSED` を `貸出不可` として表示する。
- `備品種別` は数量差を持たない粗い分類とし、`長机 2台` / `長机 3台` の差は `備品名` で表現する。
- 別権限の認証済み利用者が本画面へアクセスした場合の制御は、`docs/03_designs/security-design.md` に従う。

- 関連ユースケース：`UC-001`、`UC-002`
- 関連機能要件：`FR-001`、`FR-002`、`FR-007`
- 利用サービス：
  - 初期表示用サービス：`備品検索初期表示サービス(SAS301)`
  - 検索実行用サービス：`備品検索実行サービス(SAS302)`
  - 貸出申請開始用サービス：`貸出申請開始サービス(SAS303)`
- 関連セキュリティ設計：
  - `docs/03_designs/security-design.md`
- 関連データ：
  - `M_EQUIPMENT`
