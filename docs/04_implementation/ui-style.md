# UIスタイル規約

本書は、本リポジトリにおける業務画面のUIスタイル方針を定義する。
実装者および GitHub Copilot は、本書に従って画面の見た目を統一すること。

---

## 1. 適用範囲

- 本書は、ログイン画面を除く業務画面に適用する。
- 対象画面の共通スタイルは、`src/main/resources/static/css/business-common.css` に定義する。
- 個別画面固有の見た目が必要な場合も、まず本書および共通CSSを優先して利用する。

---

## 2. 基本方針

- 業務画面は、可読性、視認性、学習用途での分かりやすさを優先する。
- レイアウトは、ヘッダー、本文、フッターの構成を基本とする。
- 本文領域は白背景のカード状コンテナとし、背景色との差分で主要領域を明確にする。
- 色、余白、タイポグラフィは画面ごとにばらつかせず、共通CSSで統一する。

---

## 3. レイアウト規約

### 3.1 画面全体

- 画面全体の背景色は淡いグレー系とする。
- 業務画面の `body` には業務画面用のクラスを付与する。
- 本文領域は中央寄せとし、最大幅を固定して横に広がりすぎないようにする。

### 3.2 ヘッダー

- ヘッダーは濃色背景とし、システム名を明確に表示する。
- ヘッダーは全業務画面で共通化する。

### 3.3 本文

- 本文は白背景、角丸、控えめな影付きのパネルとして表示する。
- 画面タイトル、説明文、一覧、操作ボタンの順で配置する。
- セクション間には十分な上下余白を設ける。

### 3.4 フッター

- フッターはヘッダーより弱い背景色とし、補助情報のみを表示する。
- フッターも全業務画面で共通化する。

---

## 4. コンポーネント規約

### 4.1 見出し

- 画面タイトルは `h2` 相当、セクションタイトルは `h3` 相当で統一する。
- 見出しの下には過剰な装飾を付けず、余白と文字サイズ差で階層を表現する。

### 4.2 メッセージ

- ワーニングメッセージは背景色と境界線で注意喚起できるようにする。
- 情報メッセージや対象なしメッセージも、本文から視認できる余白と色分けを行う。

### 4.3 テーブル

- 一覧テーブルはヘッダー行とデータ行の境界が識別できるようにする。
- 罫線は過度に強くせず、行ごとの視認性を確保する程度に留める。
- テーブルが表示されない場合の代替文言は本文中で読める位置に表示する。

### 4.4 ボタン

- 業務画面の主要ボタンは角丸と十分な余白を持たせる。
- 未実装や操作不可のボタンは、無効状態であることが視覚的に分かるようにする。

---

## 5. CSS配置と適用方法

- 共通CSSは `src/main/resources/static/css/business-common.css` に配置する。
- ログイン画面以外の業務画面テンプレートのみ、このCSSを `link` 要素で読み込む。
- 共通フラグメントで使用するクラス名も、本CSSで定義する。
- 画面固有の見た目を追加する場合は、共通CSSの責務を崩さないようにクラスを分離する。

---

## 6. 禁止事項

- 業務画面ごとに異なる配色や余白体系を持ち込むこと
- 同等の目的を持つスタイルを各HTMLにインライン記述すること
- ログイン画面に業務画面用CSSを適用すること
- 共通コンポーネントの見た目を画面単位で無秩序に上書きすること

---
