# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS502_approve-lending-request_service`
- 名称：`申請承認サービス`

## 2. 役割と責務
- 管理者承認・却下・返却確認画面(V500) における申請承認ユースケース境界を担う。
- 入力受付、トランザクション境界管理、例外の分類を担当する。
- 実際の状態更新は `申請承認コマンドサービス(SCS010)` に委譲する。

## 3. 目的・スコープ
- 目的：承認待ちの貸出申請を承認し、申請状態を貸出中へ更新するとともに、対象備品を貸出中状態へ更新する。
- スコープ：
  - 対象画面：`管理者承認・却下・返却確認画面(V500)`
  - 対象データ：貸出申請
  - 業務範囲：申請承認ユースケースの流れ制御

## 4. 前提条件・事後条件

### 4.1 前提条件
- 認証済みの管理者であること。
- 入力値の型・必須・形式検証は Controller 層で完了している前提とする。
- 対象申請IDおよび楽観ロック用バージョンが入力されていること。

### 4.2 事後条件
- 正常終了時：
  - 対象申請の状態が `LENT` へ更新される。
  - 対象備品の状態が `LENT` へ更新される。
  - 承認者、管理者コメント、承認日時が記録される。
- 異常終了時：
  - トランザクションはロールバックされる。
  - UI へ業務例外またはシステム例外が通知される。

## 5. 処理フロー概要
1. 管理者ユーザーID、貸出申請ID、管理者コメント、バージョンを受け取る。
2. `申請承認コマンドサービス(SCS010)` に更新処理を委譲する。
3. 例外発生時は `02_architecture/error-handling.md` に従い業務例外とシステム例外を分類する。

## 6. 内部で使用するサービス
- Query Service：
  - なし
- Command Service：
  - `申請承認コマンドサービス(SCS010)`
- Pure Service：
  - なし

## 7. 入出力

### 7.1 入力
| 項目 | 型 | 必須 | 備考 |
|-----|----|-----|-----|
| adminUserId | string | ○ | 承認を行う管理者ユーザーID |
| lendingRequestId | long | ○ | 承認対象の貸出申請ID |
| reviewComment | string | 任意 | 最大500文字 |
| version | long | ○ | 楽観ロック確認用 |

### 7.2 出力
- 外部公開メソッドの戻り値は `void` とする。

## 8. 例外マッピング方針
- 業務例外：対象申請が存在しない、承認待ちでない、またはバージョン不一致の場合は更新不可として通知する。
- システム例外：永続化失敗や想定外障害は汎用エラーとして通知する。
- 参照：`02_architecture/error-handling.md`

## 9. トランザクション・整合性
- トランザクション境界は Application Service とする。
- `申請承認コマンドサービス(SCS010)` は本サービスのトランザクション境界を引き継いで実行する。
