# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS702_register-equipment_service`
- 名称：`備品登録サービス`

## 2. 役割と責務
- `管理者備品編集画面(V700)` における備品登録ユースケース境界を担う。
- 入力受付、トランザクション境界管理、例外の分類を担当する。
- 実際の備品登録は `備品登録コマンドサービス(SCS013)` に委譲する。

## 3. 目的・スコープ
- 目的：貸出管理対象となる新規備品を登録し、管理台帳へ追加する。
- スコープ：
  - 対象画面：`管理者備品編集画面(V700)`
  - 対象データ：備品
  - 業務範囲：備品登録ユースケースの流れ制御

## 4. 前提条件・事後条件

### 4.1 前提条件
- 認証済みの管理者であること。
- 入力値の型・必須・形式検証は Controller 層で完了している前提とする。

### 4.2 事後条件
- 正常終了時：
  - 新規備品が登録される。
  - システム登録日が登録当日で記録される。
  - 初期状態が指定値で設定される。
- 異常終了時：
  - トランザクションはロールバックされる。
  - UI へ業務例外またはシステム例外が通知される。

## 5. 処理フロー概要
1. 管理者ユーザーID、備品名、備品種別、保管場所、初期状態、備考を受け取る。
2. `備品登録コマンドサービス(SCS013)` に登録処理を委譲する。
3. 例外発生時は `02_architecture/error-handling.md` に従い業務例外とシステム例外を分類する。

## 6. 内部で使用するサービス
- Query Service：
  - なし
- Command Service：
  - `備品登録コマンドサービス(SCS013)`
- Pure Service：
  - なし

## 7. 入出力

### 7.1 入力
| 項目 | 型 | 必須 | 備考 |
|-----|----|-----|-----|
| adminUserId | string | ○ | 登録を行う管理者ユーザーID |
| equipmentName | string | ○ | 最大100文字 |
| equipmentType | string | ○ | 備品種別マスタで有効な候補から選択する |
| storageLocation | string | ○ | 最大100文字 |
| initialStatus | string | ○ | `AVAILABLE`、`UNAVAILABLE`、`DISPOSED` を許可する |
| remarks | string | 任意 | 最大500文字 |

### 7.2 出力
- 外部公開メソッドの戻り値は `void` とする。

## 8. 例外マッピング方針
- 業務例外：必須項目不足、重複登録、備品種別マスタ未登録、許可されない初期状態は登録不可として通知する。
- システム例外：永続化失敗や想定外障害は汎用エラーとして通知する。
- 参照：`02_architecture/error-handling.md`

## 9. トランザクション・整合性
- トランザクション境界は Application Service とする。
- `備品登録コマンドサービス(SCS013)` は本サービスのトランザクション境界を引き継いで実行する。
