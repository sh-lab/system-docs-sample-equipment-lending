# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS703_update-equipment-info_service`
- 名称：`備品情報更新サービス`

## 2. 役割と責務
- `管理者備品編集画面(V700)` における備品情報更新ユースケース境界を担う。
- 入力受付、トランザクション境界管理、例外の分類を担当する。
- 実際の備品名、状態、備考の更新は `備品情報更新コマンドサービス(SCS014)` に委譲する。

## 3. 目的・スコープ
- 目的：登録済み備品の備品名、状態、備考を業務ルールに従って更新する。
- スコープ：
  - 対象画面：`管理者備品編集画面(V700)`
  - 対象データ：備品
  - 業務範囲：備品情報更新ユースケースの流れ制御

## 4. 前提条件・事後条件

### 4.1 前提条件
- 認証済みの管理者であること。
- 入力値の型・必須・形式検証は Controller 層で完了している前提とする。
- 対象備品IDおよび楽観ロック用バージョンが入力されていること。

### 4.2 事後条件
- 正常終了時：
  - 対象備品の備品名、状態、備考が指定値へ更新される。
  - 更新者および更新日時が記録される。
- 異常終了時：
  - トランザクションはロールバックされる。
  - UI へ業務例外またはシステム例外が通知される。

## 5. 処理フロー概要
1. 管理者ユーザーID、備品ID、更新後の備品名、更新後状態、備考、バージョンを受け取る。
2. `備品情報更新コマンドサービス(SCS014)` に更新処理を委譲する。
3. 例外発生時は `02_architecture/error-handling.md` に従い業務例外とシステム例外を分類する。

## 6. 内部で使用するサービス
- Query Service：
  - なし
- Command Service：
  - `備品情報更新コマンドサービス(SCS014)`
- Pure Service：
  - なし

## 7. 入出力

### 7.1 入力
| 項目 | 型 | 必須 | 備考 |
|-----|----|-----|-----|
| adminUserId | string | ○ | 更新を行う管理者ユーザーID |
| equipmentId | long | ○ | 更新対象の備品ID |
| equipmentName | string | ○ | 更新後の備品名。最大100文字 |
| targetStatus | string | ○ | `AVAILABLE`、`UNAVAILABLE`、`DISPOSED` を許可する |
| remarks | string | 任意 | 最大500文字 |
| version | long | ○ | 楽観ロック確認用 |

### 7.2 出力
- 外部公開メソッドの戻り値は `void` とする。

## 8. 例外マッピング方針
- 業務例外：対象備品が存在しない、貸出申請中または貸出中で更新不可、許可されない遷移、またはバージョン不一致の場合は更新不可として通知する。
- システム例外：永続化失敗や想定外障害は汎用エラーとして通知する。
- 参照：`02_architecture/error-handling.md`

## 9. トランザクション・整合性
- トランザクション境界は Application Service とする。
- `備品情報更新コマンドサービス(SCS014)` は本サービスのトランザクション境界を引き継いで実行する。
