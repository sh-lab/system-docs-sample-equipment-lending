# UI文言規約

## 1. 文書の目的
本書は、本システムに表示する UI 文言の設計方針を定義するための規約である。

利用者向け表示文言の簡潔さと、メッセージ文体の統一を確保し、画面設計書および実装が同じ判断基準に従うことを目的とする。

---

## 2. 適用範囲
- 本書は、`login` を含む全画面の利用者向け表示文言に適用する。
- 対象には、画面タイトル、見出し、ボタン文言、固定ラベル、対象なしメッセージ、完了メッセージ、警告メッセージ、業務エラーメッセージを含む。
- 文言定義は `src/main/resources/i18n/labels*.properties` および `src/main/resources/i18n/message*.properties` を正とする。

---

## 3. 基本方針
- UI 文言は、意味が変わらない範囲で簡潔にする。
- 同じ意味の操作や状態は、画面ごとに不必要な言い換えをしない。
- 利用者向けメッセージは、ですます調で統一する。
- 完了、警告、エラーの各メッセージは、何が起きたかを短く示し、必要な場合のみ次の行動を案内する。
- 設計書に固定表示文言を記載する場合は、実装と一致させる。

---

## 4. 文言設計ルール
### 4.1 見出し・ラベル
- 見出しは、画面の目的がすぐ分かる表現にする。
- ラベルは、業務用語として意味が通る最短の表現を優先する。
- 同じ意味を持つラベルは、同一表現へ寄せる。

### 4.2 ボタン文言
- ボタン文言は、押下後の結果が想像できる短い動詞句にする。
- `登録する` のような補足語がなくても意味が通る場合は、省略を優先する。

### 4.3 メッセージ文言
- 完了メッセージは `〜しました。` または `〜が完了しました。` の形を基本とする。
- 警告メッセージは、注意事項と必要な再操作を短く示す。
- エラーメッセージは、失敗内容と再操作の起点を短く示す。
- 対象なしメッセージは、ですます調で簡潔に表示する。

---

## 5. 運用ルール
- 新しい画面文言を追加する場合は、本規約に従って `i18n` 定義へ追加する。
- 既存文言を変更する場合は、関連する画面設計書の固定文言例も同時に更新する。
- 文言変更時は、画面表示と既存テストに影響がないことを確認する。
