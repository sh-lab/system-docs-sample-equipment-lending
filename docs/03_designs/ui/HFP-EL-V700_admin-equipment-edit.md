# 画面設計書

## 1. 画面ID・名称
- 画面ID：`HFP-EL-V700_admin-equipment-edit`
- 画面名称：`管理者備品編集画面`

---

## 2. 概要
本画面は、管理者が `管理者備品検索画面(V600)` から遷移し、
新規備品登録または登録済み備品の備品情報更新を行うための画面である。

本画面は 1 つの画面で `新規登録モード` と `編集モード` を切り替える。
`新規登録モード` では貸出管理対象となる新規備品を登録し、
`編集モード` では登録済み備品の現在情報を確認したうえで備品名、備品状態、備考を更新する。

---

## 3. 画面レイアウト概要
- 種別：詳細・更新画面
- 主な構成：
  - 上部：画面タイトル、画面モード表示、メッセージエリア
  - 中央上段：備品基本情報
  - 中央下段：状態更新領域
  - 下部：戻るボタン、実行ボタン

### 表示方針
- 画面タイトルとして `管理者備品編集画面` を表示する。
- `新規登録モード` では、`V600` の `新規備品登録` 操作から遷移し、空の入力欄と当日を示す `システム登録日` を表示する。
- `新規登録モード` の `備品種別` は、備品種別マスタで有効な候補から選択する。
- `編集モード` では、`V600` の検索結果一覧から選択した備品を表示し、備品管理番号、備品種別、保管場所、システム登録日、現在状態を参照できるものとする。
- `編集モード` では、既存備品の識別情報のうち備品管理番号、備品種別、保管場所、システム登録日は参照表示とし、更新対象は備品名、備品状態、備考に限定する。
- `更新後状態` は、新規登録時は初期状態、編集時は更新後の備品状態として扱う。
- 実行完了後は `V600` へ戻り、完了メッセージを表示する。
- 業務上更新不可の備品に対しては、画面上部にメッセージを表示し、再選択を促す。

| 画面モード | ラベル | - | `新規登録`、`編集` | ラベル | 遷移元の操作種別で決定する |
| メッセージエリア | メッセージ | - | - | インフォ／ワーニング／エラー | 実行結果または業務エラーを表示する |

| システム登録日 | テキスト | - | - | 日付 | `新規登録モード` では登録予定日として当日を表示し、`編集モード` では登録済み日付を表示する |
| 備考 | テキストエリア | 任意 | 最大500文字 | テキストエリア | `新規登録モード` と `編集モード` のいずれでも入力可とし、`編集モード` では既存値を初期表示する |

| CSRFトークン | 隠し項目 | 必須 | フレームワーク生成値 | hidden | 更新系操作で送信する |

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | `V600` からの画面遷移時 | 画面モードと対象備品IDに応じて表示情報を取得し、入力可否を切り替えて表示する | `管理者備品編集画面初期表示サービス(SAS701)` |
| 備品登録実行 | `[登録する]` ボタンクリック | 入力内容を用いて新規備品を登録し、完了後は `V600` へ戻る | `備品登録サービス(SAS702)` |
| 備品情報更新実行 | `[更新する]` ボタンクリック | 対象備品の備品名、状態、備考を更新し、完了後は `V600` へ戻る | `備品情報更新サービス(SAS703)` |
| 管理者備品検索へ戻る | `[管理者備品検索へ戻る]` ボタンクリック | `V600` へ戻る | `－（画面遷移のみ）` |

## 6. バリデーション方針
### クライアント側
- `新規登録モード` では、`備品名`、`備品種別`、`保管場所`、`更新後状態` を入力必須とする。
- `備品名` は最大100文字、`保管場所` は最大100文字、`備考` は最大500文字とし、超過入力は受け付けない。
- `更新後状態` は `貸出可能`、`貸出不可`、`廃棄` の候補から選択する。
- `備品種別` は備品種別マスタで有効な候補からのみ選択でき、画面上で候補追加・変更は行わない。
- 実行ボタン押下後は、二重送信防止のため一時非活性化する。

### サーバ側
- 本画面は `ADMIN` 権限を持つ認証済み利用者を利用対象とし、認証・認可の具体設計は `docs/03_designs/security-design.md` に従う。
- Controller はリクエスト受信時に、`備品名` の必須・最大100文字、`保管場所` の必須・最大100文字、`備考` の最大500文字について基本バリデーションを行う。
- `新規登録モード` の初期表示では、備品種別マスタから有効な備品種別候補と初期状態候補、および当日の日付を取得する。
- `編集モード` の初期表示では、対象備品IDが存在することを確認し、対象備品の現在情報と `VERSION` を取得する。
- `備品登録サービス(SAS702)` は、同一備品を識別できなくなるような不正な登録を防止し、指定された備品種別が備品種別マスタに存在することを確認したうえで、システム登録日を登録当日で確定させる。
- `備品情報更新サービス(SAS703)` は、対象備品が存在し、`VERSION` が一致し、かつ状態遷移ルールに違反しないことを確認したうえで更新する。
- `貸出申請中` および `貸出中` の備品に対する更新は業務例外として扱い、「貸出申請中と貸出中の備品状態は更新できません」を表示する。

## 7. エラー・メッセージ表示方針
- 備品登録成功時：`V600` でインフォ「備品を登録しました。」を表示する。
- 備品情報更新成功時：`V600` でインフォ「備品情報を更新しました。」を表示する。
- 必須項目不足時：画面上部に入力エラーを表示し、入力の見直しを促す。
- 対象備品が削除済みまたは参照不可の場合：画面上部に業務メッセージを表示し、`V600` での再選択を促す。
- 対象備品が貸出申請中または貸出中で更新不可の場合：画面上部に「貸出申請中と貸出中の備品状態は更新できません」を表示し、更新を行わない。
- 楽観ロック不一致時：画面上部に業務メッセージを表示し、最新状態の再確認を促す。
- システム例外：汎用エラー画面へ遷移する。

`02_architecture/error-handling.md` に従い、UI 層では業務例外のみユーザー通知し、システム例外は汎用エラーとして扱う。

## 8. 非機能要件への配慮
- 操作性：管理者備品検索から新規登録と状態更新を同一画面で扱い、管理作業の導線を単純化する。
- 一貫性：`新規登録モード` と `編集モード` で画面構造を共通化し、学習コストを抑える。
- 重複操作防止：実行ボタン押下後は二重送信を防止し、同一備品への重複更新を抑止する。
- アクセシビリティ：キーボード操作のみで入力、状態選択、更新実行、戻る操作へ到達できる構成とする。
- セキュリティ：更新系操作は CSRF トークン送信を前提とし、管理者権限を持たない利用者は対象外とする。

## 9. 備考
- 本画面は管理者専用であり、利用者は対象外である。
- `新規登録モード` では、備品管理番号は登録時に採番される前提とする。
- `編集モード` では、備品種別、保管場所、システム登録日の変更は扱わず、備品名、備品状態、備考の更新に限定する。
- 備品種別マスタ自体の追加・変更・削除は、本画面の対象外とする。
- `システム登録日` は登録時に確定し、その後は更新しない。
