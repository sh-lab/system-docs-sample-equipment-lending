# 画面設計書

## 1. 名称
- 画面名称：`多重送信エラー画面`
- 本画面は画面ID体系の適用対象外とする。

---

## 2. 概要
本画面は、重複送信防止用ワンタイムトークンが不正または再利用済みであることを検知した場合に表示する専用エラー画面である。

利用者には元画面を再利用させず、ログイン中の利用者区分に応じたマイページへ戻して対象画面を開き直させることを目的とする。

---

## 3. 画面レイアウト概要
- 種別：メッセージ画面
- 主な構成：
  - 上部：画面タイトル
  - 中央：エラーメッセージ、案内文
  - 下部：マイページへ戻るボタン

### レイアウトイメージ
```text
+--------------------------------------------------------------------+
|                        多重送信エラー                              |
+--------------------------------------------------------------------+
| [!] 画面を再表示してからやり直してください。                       |
|                                                                    |
| 同じ操作が繰り返し送信された可能性があります。                     |
| マイページに戻り、画面を開き直してからやり直してください。         |
|                                                                    |
|                                      [マイページへ戻る]            |
+--------------------------------------------------------------------+
```

### 表示方針
- 画面は利用者向け・管理者向けで分割せず、1画面共通とする。
- 表示するエラーメッセージは重複送信検知専用の固定メッセージとする。
- `マイページへ戻る` の遷移先のみ、ログイン中の利用者区分に応じて `V100` または `V200` に切り替える。
- 元画面へ戻る導線は提供しない。

---

## 4. 画面項目定義
| 項目名 | 種別 | 必須 | 入力制約 | 表示形式 | 備考 |
|------|------|------|---------|---------|------|
| 画面タイトル | ラベル | - | - | テキスト | `多重送信エラー` を固定表示する |
| エラーメッセージ | メッセージ | 必須 | 固定メッセージID | エラー表示 | `MSG_E_011` を表示する |
| 案内文 | テキスト | 必須 | 固定文言 | テキスト | マイページへ戻って画面を開き直すことを案内する |
| マイページへ戻る | ボタン | 必須 | - | ボタン | 利用者区分に応じて `V100` または `V200` へ遷移する |

---

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | `HandlerInterceptor` からのリダイレクト | 固定メッセージと戻り先を組み立てて表示する | `－（画面表示のみ）` |
| マイページへ戻る | `[マイページへ戻る]` ボタンクリック | ログイン中の利用者区分に応じたマイページへ遷移する | `－（画面遷移のみ）` |

---

## 6. バリデーション方針
### クライアント側
- 入力項目は持たない。

### サーバ側
- 本画面は認証済み利用者を利用対象とし、認証・認可の具体設計は `docs/03_designs/security-design.md` に従う。
- 戻り先のマイページは、認証コンテキスト上の利用者区分に基づいて決定する。
- 個別画面の入力値や遷移元パラメータに依存して戻り先を判定しない。

---

## 7. エラー・メッセージ表示方針
- ワンタイムトークンが不正または再利用済みである場合：本画面を表示し、`MSG_E_011` を表示する。
- 未認証で本画面へアクセスした場合：ログイン画面へ誘導する。
- システム例外：汎用エラー画面へ遷移する。

`02_architecture/error-handling.md` に従い、UI 層では業務上通知すべき内容のみ本画面で扱い、システム例外は汎用エラーとして扱う。

---

## 8. 非機能要件への配慮
- 操作性：再送不可であることを明示し、次に取るべき行動をマイページ復帰へ限定する。
- 一貫性：対象機能ごとに別画面を作らず、重複送信検知時の導線を共通化する。
- セキュリティ：重複送信検知時に元画面の入力再利用や再送導線を残さない。
- 保守性：戻り先判定は利用者区分に集約し、画面ごとの個別分岐を持ち込まない。

---

## 9. 備考
- 本画面は `V400`、`V500`、`V700` の更新系 POST に共通適用する。
- 重複送信検知の契機は `HandlerInterceptor` によるワンタイムトークン検証失敗である。
- 利用者区分ごとのホーム画面定義は `docs/03_designs/security-design.md` に従う。

- 関連機能要件：`FR-007`
- 関連セキュリティ設計：
  - `docs/03_designs/security-design.md`
- 画面遷移：
  - 遷移元：`利用者貸出申請・返却画面(V400)`
  - 遷移元：`管理者承認・却下・返却確認画面(V500)`
  - 遷移元：`管理者備品編集画面(V700)`
  - 遷移先：`利用者マイページ(V100)`
  - 遷移先：`管理者マイページ(V200)`
