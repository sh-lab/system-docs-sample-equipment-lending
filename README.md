# system-docs-sample-equipment-lending

本リポジトリは、業務システム開発向けテンプレートの利用例として作成した、
組織内利用を想定した備品貸出管理システムの**学習用サンプル**である。

要件定義・設計を正とした開発プロセス、および業務システムにおける
責務分離（画面・アプリケーションサービス・コマンドサービス・ピュアサービス）の考え方を、
具体例を通して示すことを目的としている。

---

## ステータス

- **Status:** Beta（設計書・サンプルアプリ・学習用コンテンツ公開済）
- 本リポジトリでは、要件定義書・設計書および、それに基づく動作可能なサンプルアプリを公開している。
- あわせて、学習用の読み物コンテンツおよび演習課題（設計変更・機能追加を伴う課題）を追加した。
- 演習課題および読み物は、設計を起点とした開発プロセスを段階的に学べるよう構成されており、今後も必要に応じて拡張される可能性がある。
- 本リポジトリは業務システム開発を学ぶための教材を目的としており、商用利用を前提とした完成品ではない。
- 設計・構成・粒度は、教材としての改善や内容拡充に伴い、今後変更される可能性がある。

## 本リポジトリの特徴

- 要件定義・設計書を正とした構成
- `docs-template/**` を参照テンプレートとし、`docs/**` に実体を配置
- 画面用アプリケーションサービスと業務コマンドの明確な分離
- 業務ルールをピュアサービスとして分離
- フレームワーク依存を設計書に持ち込まない方針

> **アーキテクチャについて**:
> 本サンプルはトランザクションスクリプト方式を採用している
> （詳細は [ADR-001](docs/99_adr/ADR-001_transaction-script-adoption.md) を参照）。
> これは業務ルールが比較的単純な本サンプルの特性と、
> 学習容易性を優先した選択であり、
> すべてのプロジェクトに適用すべき汎用的な推奨ではない。
> プロジェクトの規模や業務ドメインの複雑さに応じて、
> 適切なアーキテクチャを選択されたい。

## ディレクトリ構成

```txt
. 
├─ docs/               # 要件定義書・設計書（正） 
├─ docs-template/      # 参照用テンプレート（変更禁止） 
├─ learning/           # 学習用コンテンツ 
└─ src/                # 実装（設計書に従って作成）
```

---

## 注意事項

- 本リポジトリは**学習用サンプル**であり、設計および実装の完全性を保証するものではない。
- ベータ版のため、設計書のID、粒度、責務分割は変更される場合がある。
- 実装は必ず `docs/**` に記載された設計書に従うことを前提としている。
- 本サンプルの認証機能は学習用に簡易化されている
  （全ユーザー共通パスワード `pass`、H2コンソール有効）。
  **本番環境でこの構成を使用してはならない。**
