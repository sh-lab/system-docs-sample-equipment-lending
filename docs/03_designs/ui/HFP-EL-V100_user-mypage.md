# 画面設計書

## 1. 画面ID・名称
- 画面ID：`HFP-EL-V100_user-mypage`
- 画面名称：`利用者マイページ`

---

## 2. 概要
本画面は、利用者が自身に関連する貸出申請を一覧で確認し、対象の申請詳細画面または備品検索画面へ遷移するための画面である。

貸出中の申請、承認待ちまたは返却確認待ちの申請、却下済みで未確認の申請を表示対象とし、一覧画面では業務操作を実行せず、詳細確認および次画面への導線提供を責務とする。

却下済みで未確認の申請が 1 件以上存在する場合は、利用者が確認漏れに気づけるよう、画面上部にワーニングメッセージ「却下された申請があります。」を表示する。

---

## 3. 画面レイアウト概要
- 種別：一覧画面
- 主な構成：
  - 上部：画面タイトル
  - タイトル下：ワーニングメッセージ
  - 中央上段：貸出中一覧
  - 中央下段：申請中一覧
  - 下部：備品検索ボタン

### レイアウトイメージ
```text
+------------------------------------------------+
|                利用者マイページ                |
+------------------------------------------------+
| [!] 却下された申請があります。                 |
| ※ 却下申請が存在する場合のみ表示              |
|                                                |
|                                                |
| 貸出中一覧                                     |
| +--------+------------------+----------+       |
| | 詳細   | 備考             | 状態     |       |
| +--------+------------------+----------+       |
| | [詳細] | ○○のため申請     | 貸出中   |       |
| | [詳細] | △△用途          | 貸出中   |       |
| +--------+------------------+----------+       |
|                                                |
| 申請中一覧                                     |
| +--------+------------------+------------------+--------------+ |
| | 詳細   | 備考             | 管理者コメント   | 状態         | |
| +--------+------------------+------------------+--------------+ |
| | [詳細] | ○○のため申請     |                  | 承認待ち     | |
| | [詳細] | △△用途          | 用途が不明である | 却下         | |
| +--------+------------------+------------------+--------------+ |
|                                                |
| [備品検索]                                     |
+------------------------------------------------+
```

### 表示方針
- 却下済みで未確認の申請が 1 件以上存在する場合、画面タイトル直下にワーニングメッセージ「却下された申請があります。」を表示する。
- 貸出中一覧には、利用者本人が現在貸出中である申請を表示する。
- 申請中一覧には、`承認待ち`、`返却確認待ち`、`却下` の利用者が確認対象とすべき申請を表示する。
- `却下` の申請では、利用者が詳細画面で却下理由および管理者コメントを確認できるよう、管理者コメント列を表示する。
- 申請に対する返却申請、却下確認などの業務操作は、詳細画面へ遷移して実施する。

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | 画面表示時 | 利用者本人の貸出中一覧および申請中一覧を取得して表示する。却下済みで未確認の申請が 1 件以上存在する場合は、画面上部へワーニングメッセージを表示する | `利用者マイページ初期表示サービス(SAS101)` |
| 貸出中一覧詳細表示 | 貸出中一覧の `[詳細]` ボタンクリック | 選択した申請の詳細画面へ遷移する | `－（画面遷移のみ）` |
| 申請中一覧詳細表示 | 申請中一覧の `[詳細]` ボタンクリック | 選択した申請の詳細画面へ遷移する | `－（画面遷移のみ）` |
| 備品検索開始 | `[備品検索]` ボタンクリック | 備品検索画面へ遷移する | `－（画面遷移のみ）` |

## 6. バリデーション方針
### クライアント側
- 本画面は一覧表示と画面遷移が中心であり、自由入力項目は持たない。
- 画面表示時に対象一覧が 0 件であってもエラーとはせず、対象データが存在しない旨を画面上に表示する。
- 一覧の `[詳細]` ボタンは、対象申請IDを保持できる行に対してのみ活性表示する。

### サーバ側
- 本画面は `USER` 権限を持つ認証済み利用者を利用対象とし、認証・認可の具体設計は `docs/03_designs/security-design.md` に従う。
- 一覧取得時は、認証済み利用者本人の申請のみを表示対象とする。
- 指定された申請IDが利用者本人に紐づかない場合は、業務例外として扱う。
- `申請中一覧` の抽出対象状態は `PENDING_APPROVAL`、`PENDING_RETURN_CONFIRMATION`、`REJECTED` とする。
- `貸出中一覧` の抽出対象状態は `LENT` とする。
- 一覧取得処理は画面から直接行わず、サービス層を経由して必要な参照処理へ委譲する。

---

## 7. エラー・メッセージ表示方針
- 却下済みで未確認の申請が存在する場合：画面上部にワーニング「却下された申請があります。」を表示する。
- 一覧対象が存在しない場合：各一覧領域に「表示できる申請はありません。」を表示する。
- 業務例外：画面上部にメッセージを表示し、利用者に再操作または再読み込みを促す。
- システム例外：汎用エラー画面へ遷移する。

`02_architecture/error-handling.md` に従い、UI層では業務例外のみユーザー通知し、システム例外は汎用エラーとして扱う。

---

## 8. 非機能要件への配慮
- 表示性能：貸出中一覧および申請中一覧は、利用者本人に必要なデータのみを取得し、不要な詳細情報は含めない。
- 操作性：一覧行の主要操作は `[詳細]` のみとし、操作対象を明確にする。
- 重複操作防止：画面遷移ボタン押下中は連打による二重遷移を抑止する。
- アクセシビリティ：キーボード操作のみで各一覧の詳細ボタンおよび備品検索ボタンへ到達できる構成とする。
