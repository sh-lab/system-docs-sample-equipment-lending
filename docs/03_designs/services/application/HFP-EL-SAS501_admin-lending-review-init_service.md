# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS501_admin-lending-review-init_service`
- 名称：`管理者承認・却下・返却確認画面初期表示サービス`

## 2. 役割と責務
- 管理者承認・却下・返却確認画面(V500) の初期表示ユースケース境界を担う。
- `V200` で選択済みの貸出申請 1 件の詳細取得を Query Service に委譲する。
- UI から Query Service を直接呼び出させず、初期表示の入口を一本化する。

## 3. 目的・スコープ
- 目的：`V500` 初期表示に必要な選択中申請詳細と対象備品一覧を返却する。
- スコープ：
  - 対象画面：`管理者承認・却下・返却確認画面(V500)`
  - 対象データ：選択申請詳細、対象備品一覧
  - 業務範囲：初期表示用参照

## 4. 前提条件・事後条件

### 4.1 前提条件
- 認証済みの管理者であること。
- `adminUserId` の型・必須検証は Controller 層で完了している前提とする。
- `lendingRequestId` は `V200` で選択済みの対象申請IDとして必須である。

### 4.2 事後条件
- 正常終了時：
  - `V500` 表示に必要な選択中申請詳細を返却する。
- 異常終了時：
  - 更新処理は行われない。
  - UI へエラーが通知される。

## 5. 処理フロー概要
1. 管理者ユーザーIDと選択対象申請IDを受け取る。
2. 選択中申請詳細の取得処理を呼び出す。
3. 取得結果をそのまま UI へ返却する。

## 6. 内部で使用するサービス
- Query Service：
  - `管理者承認・却下・返却確認情報取得サービス(SQS501)`
- Command Service：
  - なし
- Pure Service：
  - なし

## 7. 入出力DTO

### 7.1 入力DTO
| 項目 | 型 | 必須 | 備考 |
|-----|----|-----|-----|
| adminUserId | string | ○ | 管理者ユーザーID |
| lendingRequestId | long | ○ | `V200` で選択された貸出申請ID |

### 7.2 出力DTO
| 項目 | 型 | 必須 | 説明 |
|-----|----|-----|-----|
| selectedRequest | `FindAdminLendingReviewQueryServiceImpl.Detail` | ○ | 選択中申請詳細 |
| mode | string | ○ | `APPROVAL_REVIEW` または `RETURN_CONFIRM` |

## 8. 例外マッピング方針
- 業務例外：指定申請IDが存在しない、または表示対象状態でない場合は `V200` での再選択を促す。
- システム例外：汎用エラーとして通知する。
- 参照：`02_architecture/error-handling.md`

## 9. トランザクション・整合性
- 参照専用サービスであり、状態変更は行わない。
- 取得対象は単一申請の詳細に限定する。
