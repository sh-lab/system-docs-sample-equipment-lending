# 画面設計書

## 1. 画面ID・名称
- 画面ID：`HFP-EL-V200_admin-mypage`
- 画面名称：`管理者マイページ`

---

## 2. 概要
本画面は、管理者が承認待ち申請一覧および返却確認待ち申請一覧を確認し、
対象申請の詳細確認画面である `管理者承認・却下・返却確認画面(V500)` へ遷移するための入口画面である。

本画面では一覧の確認と対象選択のみを扱い、申請承認、申請却下、返却確認の状態更新は行わない。
業務更新はすべて `管理者承認・却下・返却確認画面(V500)` で実施する。

---

## 3. 画面レイアウト概要
- 種別：一覧・導線画面
- 主な構成：
  - 上部：共通ヘッダ、画面タイトル、メッセージエリア
  - 中央上段：承認待ち申請一覧
  - 中央下段：返却確認待ち申請一覧
  - 下部：共通フッタ

### レイアウトイメージ
```text
+------------------------------------------------------------------------------------------------+
|                                      管理者マイページ                                          |
+------------------------------------------------------------------------------------------------+
| [i] 申請を承認しました。                                                                       |
| [i] 申請を却下しました。                                                                       |
| [i] 返却確認が完了しました。                                                                   |
| ※ 実行結果メッセージは状態に応じて表示                                                         |
|                                                                                                |
| 承認待ち申請一覧                                                                                |
| +--------+------------+------------+----------------------+-------------------+--------------+ |
| | 詳細   | 申請ID     | 申請者     | 申請コメント         | 申請日時          | 状態         | |
| +--------+------------+------------+----------------------+-------------------+--------------+ |
| | [確認] | 2001       | USER02     | 長机を申請する。     | 2026-01-10 09:00  | 承認待ち     | |
| +--------+------------+------------+----------------------+-------------------+--------------+ |
|                                                                                                |
| 返却確認待ち申請一覧                                                                            |
| +--------+------------+------------+----------------------+-------------------+--------------+ |
| | 詳細   | 申請ID     | 申請者     | 返却申請コメント     | 返却申請日時      | 状態         | |
| +--------+------------+------------+----------------------+-------------------+--------------+ |
| | [確認] | 2003       | USER02     | 返却したので確認依頼 | 2026-01-14 17:00  | 返却確認待ち | |
| +--------+------------+------------+----------------------+-------------------+--------------+ |
+------------------------------------------------------------------------------------------------+
```

### 表示方針
- 画面タイトルとして `管理者マイページ` を表示する。
- 上部メッセージエリアには、`V500` での承認・却下・返却確認完了後にリダイレクトされた実行結果メッセージを表示できるものとする。
- `承認待ち申請一覧` には、状態が `PENDING_APPROVAL` の申請のみを表示する。
- `返却確認待ち申請一覧` には、状態が `PENDING_RETURN_CONFIRMATION` の申請のみを表示する。
- 各一覧は、`詳細` 操作から `管理者承認・却下・返却確認画面(V500)` へ遷移できる。
- 一覧では業務更新を行わず、対象の選択と状況確認に限定する。
- 一覧が空の場合は、対象なしメッセージ「表示できる申請はありません。」を表示する。

## 5. イベント・操作定義
| イベント | トリガー | 処理内容 | 呼び出し先 |
|--------|----------|----------|------------|
| 初期表示 | 画面表示時 | 管理者向け一覧表示データを取得し、承認待ち申請一覧および返却確認待ち申請一覧を表示する | `管理者マイページ初期表示サービス(SAS201)` |
| 承認待ち申請の詳細表示 | `承認待ち申請一覧.詳細` ボタンクリック | 対象申請IDを指定して `V500` へ遷移する | `－（画面遷移のみ）` |
| 返却確認待ち申請の詳細表示 | `返却確認待ち申請一覧.詳細` ボタンクリック | 対象申請IDを指定して `V500` へ遷移する | `－（画面遷移のみ）` |

## 6. バリデーション方針
### クライアント側
- 本画面は参照と遷移のみを扱い、自由入力項目は持たない。
- 一覧に表示対象が存在する場合のみ `詳細` ボタンを表示する。

### サーバ側
- 本画面は `ADMIN` 権限を持つ認証済み利用者を利用対象とし、認証・認可の具体設計は `docs/03_designs/security-design.md` に従う。
- 一覧取得および後続画面への遷移は、認証済み管理者本人の操作として扱う。
- `管理者マイページ情報取得サービス(SQS201)` は、承認待ち申請と返却確認待ち申請のみを取得する。
- 対象申請詳細への遷移時は、対象申請IDが `V500` で再検証される前提とする。

---

## 7. エラー・メッセージ表示方針
- 承認完了後：インフォ「申請を承認しました。」を表示する。
- 却下完了後：インフォ「申請を却下しました。」を表示する。
- 返却確認完了後：インフォ「返却確認が完了しました。」を表示する。
- 対象申請が更新済みで `V500` に表示できない場合：業務エラーを表示し、一覧の再確認を促す。
- 一覧表示対象が存在しない場合：対象なしメッセージ「表示できる申請はありません。」を表示する。
- システム例外：汎用エラー画面へ遷移する。

`02_architecture/error-handling.md` に従い、UI 層では業務例外のみユーザー通知し、システム例外は汎用エラーとして扱う。

---

## 8. 非機能要件への配慮
- 操作性：承認待ち申請一覧と返却確認待ち申請一覧を分離し、管理者が処理対象の種類を即座に判別できる構成とする。
- 表示性能：一覧には詳細確認に必要なサマリ項目のみ表示し、応答時間 1 秒以内の目標に配慮する。
- アクセシビリティ：見出し、表形式、ボタンによりキーボード操作のみで対象申請の選択へ到達できる構成とする。
- 一貫性：本画面では状態更新を行わず、業務更新は `V500` に集約する。

---

## 9. 備考
- 本画面は管理者専用であり、利用者は対象外である。
- `V200` は一覧確認の入口画面であり、承認・却下・返却確認は `管理者承認・却下・返却確認画面(V500)` で実施する。
- `V500` で処理完了後は、本画面へリダイレクトして一覧の再確認を行う前提とする。
- 別権限の認証済み利用者が本画面へアクセスした場合の制御は、`docs/03_designs/security-design.md` に従う。

- 関連ユースケース：`UC-003`、`UC-005`
- 関連機能要件：`FR-003`、`FR-005`、`FR-007`
- 利用サービス：
  - Application Service：`管理者マイページ初期表示サービス(SAS201)`
  - Query Service：`管理者マイページ情報取得サービス(SQS201)`
- 関連セキュリティ設計：
  - `docs/03_designs/security-design.md`
- 画面遷移：
  - 遷移元：`login`
  - 遷移先：`管理者承認・却下・返却確認画面(V500)`
