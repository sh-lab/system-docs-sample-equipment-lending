# アプリケーションサービス設計書（Application Service）

## 1. サービスID・名称
- ID：`HFP-EL-SAS401_user-lending-request-initialize_service`
- 名称：`利用者貸出申請・返却画面初期表示サービス`

## 2. 役割と責務
- 「利用者貸出申請・返却画面」初期表示時に、遷移元や申請状態に応じて画面モード・申請概要・対象備品一覧・入力可能項目を組み立てて返却する。
- トランザクション境界の管理と例外の分類・変換を行う。
- UIは本サービスのみを呼び出す。

## 3. 目的・スコープ
- 目的：利用者が貸出申請・返却申請・却下確認を行う画面の初期表示に必要な情報を提供する。
- スコープ：
  - 対象エンティティ：T_LENDING_REQUEST, T_LENDING_REQUEST_DETAIL, M_EQUIPMENT
  - 業務範囲：申請概要・備品一覧・コメント等の取得
  - 画面：HFP-EL-V400_user-lending-request

## 4. 前提条件・事後条件
### 4.1 前提条件
- 利用者が認証済みであること
- 入力検証（ID形式・必須）はController層で完了済み

### 4.2 事後条件
- 正常終了時：画面初期表示用DTOを返却
- 異常終了時：業務例外またはシステム例外としてUIへ通知

## 5. 処理フロー概要
1. 入力DTO（申請IDまたは備品IDリスト、遷移元等）を受け取る
2. 遷移元・申請状態に応じて画面モードを判定
3. 申請概要・対象備品一覧・コメント等を取得
4. バリデーション（権限・状態・重複・存在確認等）
5. 出力DTOを組み立てて返却
6. 例外発生時は分類しUIへ通知

## 6. 内部で使用するサービス
- Query Service：申請・備品情報取得、状態・権限チェック
- Pure Service：画面モード判定、表示用データ組み立て

## 7. 入出力DTO
### 7.1 入力DTO
| 項目 | 型 | 必須 | 備考 |
|------|----|------|------|
| userId | string | ○ | 利用者ID（認証情報） |
| requestId | string | △ | 申請ID（返却・却下確認時） |
| equipmentIds | string[] | △ | 備品IDリスト（貸出申請時） |
| from | string | ○ | 遷移元（V100:マイページ, V300:備品検索） |

### 7.2 出力DTO
| 項目 | 型 | 必須 | 説明 |
|------|----|------|------|
| mode | string | ○ | 画面モード（貸出申請/返却申請/却下確認） |
| requestSummary | object | ○ | 申請概要（ID, 状態, 日時等） |
| equipmentList | object[] | ○ | 対象備品一覧 |
| comment | string | △ | 申請コメント |
| returnComment | string | △ | 返却申請コメント |
| adminComment | string | △ | 管理者コメント |
| version | int | ○ | 楽観ロック用 |

## 8. 例外マッピング方針
- 業務例外：ユーザー向けに内容を通知（例：権限違反、状態不整合、重複、存在しない等）
- システム例外：汎用エラーとして通知

## 9. トランザクション・整合性
- トランザクション境界は本サービス単位
- 1ユースケース=1トランザクション
- 分離レベルはRead Committed
